## Common trap when writing Slice copy

Slice is often described as *"Perspectives but modern."* That's accurate but underselling. The message that actually converts is:

> *"Everything Perspectives did — plus the capabilities it never had."*

The migration pain is low. The capability upside is high. Lead with the upside (cross-drill, dynamic columns, hierarchy DBRW), then reassure on compatibility.

Avoid describing Slice as a "replacement for Perspectives" — that frames it as a forced upgrade. Frame as a successor / evolution.

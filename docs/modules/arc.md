## Primary persona — Developer

TM1 developers, consultants, and engineers. Anyone who writes TI, builds rules, manages cubes and dimensions, or deploys changes.

See `strategy/personas.md` → Developer for full profile.

**Secondary personas:**
- **Administrator** (light use — user/security management, log access, settings)
- **Consultant / Partner** — professional services teams that install Arc for multiple clients and need to switch instances fast.

## Top hooks — priority order

1. **"No more Notepad++."** Modern editor with syntax highlighting, autocomplete, find-and-replace across the whole model.
2. **"Step through TI code line by line."** Breakpoints, variable inspection, conditional pauses — a real debugger.
3. **"Manage 70 instances from one window."** Multi-server management — the Deutsche Bahn use case.
4. **"Free for 90 days. No credit card."** Zero-friction trial.

## Common trap when writing Arc copy

The temptation is to lead with the features Cubewise is proudest of engineering-wise — the multi-server architecture, the template engine. **Don't.**

A Developer who lands on `/arc/` wants to feel two things in five seconds:
1. *"This isn't Notepad++."*
2. *"I could use this tomorrow."*

Lead with the debugger and the editor. Architecture stories come later.

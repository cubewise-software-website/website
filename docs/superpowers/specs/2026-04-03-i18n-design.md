# i18n Design — Cubewise Website

**Date:** 2026-04-03
**Status:** Approved

## Overview

Add multilingual support to the Cubewise website. English is the primary language; French (fr), German (de), and Chinese Simplified (zh-hans) are the three additional locales.

**Scope:** Hand-coded pages only. Confluence-sourced blog posts and docs remain English.

---

## Architecture

**Approach: Build-time page duplication with JSON translation files.**

The build script reads each source page, substitutes translated text for marked strings, and writes a full copy of the page per locale into `dist/{locale}/`. English pages remain at their existing paths (e.g. `/about/`). Translated pages live at path-prefix URLs (e.g. `/fr/about/`, `/de/about/`, `/zh-hans/about/`).

No client-side translation library is used. Output is purely static, fully cacheable, and SEO-friendly.

---

## File Structure

```
src/
  i18n/
    en.json          # English strings — source of truth, keyed by data-i18n attribute
    fr.json          # French translations
    de.json          # German translations
    zh-hans.json     # Chinese Simplified translations
  pages/             # Existing HTML, with data-i18n attributes added
  templates/
    base.html        # Language switcher added to announcement bar

scripts/
  extract-strings.js # Scans pages for data-i18n attributes → writes src/i18n/en.json
  translate.js       # Calls Claude API to fill fr/de/zh-hans.json from en.json
  build.js           # Updated to emit dist/{locale}/ for each locale
  preview.js         # Updated to emit locale copies during local preview

dist/
  about/             # English (unchanged)
  fr/about/          # French
  de/about/          # German
  zh-hans/about/     # Chinese Simplified
  (same pattern for all pages)
```

## Marking Strings in HTML

Translatable text is marked with a `data-i18n` attribute whose value is a dot-namespaced key. The element's existing text content is the English value and remains in the HTML as the fallback.

```html
<!-- Before -->
<h1>About Cubewise</h1>
<p>We are IBM Planning Analytics experts...</p>

<!-- After -->
<h1 data-i18n="about.heading">About Cubewise</h1>
<p data-i18n="about.intro">We are IBM Planning Analytics experts...</p>
```

**Key namespaces:**

| Namespace | Used for |
|-----------|----------|
| `nav.*` | Navigation labels (Platform, Open Source, Services, Contact, etc.) |
| `footer.*` | Footer links and tagline |
| `announce.*` | Announcement bar text and utility links |
| `about.*` | About page |
| `arc.*` | Arc product page |
| `arc-plus.*` | Arc+ product page |
| `pulse.*` | Pulse product page |
| `slice.*` | Slice product page |
| `atmosphere.*` | Atmosphere product page |
| `powerconnect.*` | PowerConnect product page |
| `platform.*` | Platform overview page |
| `pricing.*` | Pricing page |
| `contact.*` | Contact page |
| `partners.*` | Partners page |
| `home.*` | Homepage |
| `open-source.*` | Open Source landing page |

**What gets a key:** Visible user-facing text — headings, body copy, button labels, link text, alt text, nav labels, footer links, `<title>` tags, meta descriptions. Do not mark: URLs, CSS class names, IDs, code snippets, brand names (Cubewise, Arc, Arc+, Pulse, Slice, Atmosphere, PowerConnect, IBM, IBM Planning Analytics).

**Attributes with translatable text:** Where text lives in an attribute (e.g. `placeholder`, `aria-label`, `alt`, `title`), use `data-i18n-attr` with a JSON value:

```html
<input data-i18n-attr='{"placeholder":"contact.email-placeholder"}' placeholder="Your email address">
```

## Translation JSON Format

```json
{
  "nav.platform": "Platform",
  "nav.contact": "Contact",
  "about.heading": "About Cubewise",
  "about.intro": "We are IBM Planning Analytics experts..."
}
```

- One flat object per locale file (no nesting)
- Keys match `data-i18n` values exactly
- Missing keys fall back to English text already in the HTML

## Scripts

### `scripts/extract-strings.js`

- Scans `src/pages/**/*.html` and `src/templates/base.html`
- Finds all elements with `data-i18n` and `data-i18n-attr`
- Writes `src/i18n/en.json` with key → English text pairs
- Safe to re-run: adds new keys, preserves existing values
- Run once initially, then whenever new `data-i18n` attributes are added

### `scripts/translate.js`

CLI: `node scripts/translate.js --locale fr`

- Reads `src/i18n/en.json`
- Reads existing `src/i18n/{locale}.json` (if present)
- Sends only missing/new keys to Claude API (`claude-sonnet-4-6`) in batches
- Provides brand context in the system prompt: Cubewise is a professional IBM Planning Analytics consultancy — tone is expert, clear, and never casual
- Writes updated `src/i18n/{locale}.json`
- Requires `ANTHROPIC_API_KEY` in `.env`

### `scripts/build.js` (updated)

For each locale (`fr`, `de`, `zh-hans`):
1. Clone each source page HTML
2. Set `<html lang="{locale}">` 
3. For each element with `data-i18n`, replace text content with the translated value (fall back to English if key missing)
4. For each element with `data-i18n-attr`, replace the specified attribute value
5. Inject `<link rel="alternate" hreflang>` tags into `<head>` for all locales
6. Set the current locale's language switcher entry as active
7. Write to `dist/{locale}/{page-path}/index.html`

The same logic applies to `scripts/preview.js` for local development.

## Language Switcher

Added to the right side of the announcement bar in `src/templates/base.html`, after the existing utility links (About, Partners, Support), separated by a vertical divider.

**Appearance:** Globe icon (inline SVG, `currentColor` via `color: var(--sand-gold)`) + current locale label (EN / FR / DE / 中文) + chevron. Clicking opens a dropdown listing all four languages.

**Dropdown entries** (example for `/about/`):
- English → `/about/`
- Français → `/fr/about/`
- Deutsch → `/de/about/`
- 中文 (简体) → `/zh-hans/about/`

The current page path (e.g. `/about/`) is injected at build time as a `data-page-path` attribute on the switcher element. The dropdown JS prefixes the locale code to resolve the correct URL per language. If a translated page doesn't exist, the switcher falls back to `/{locale}/` (the locale homepage).

**Implementation:** Small inline `<script>` in `base.html` handles open/close and link generation. No separate JS file needed.

## SEO

Each generated page includes hreflang alternate link tags:

```html
<link rel="alternate" hreflang="en"      href="https://cubewise.com/about/" />
<link rel="alternate" hreflang="fr"      href="https://cubewise.com/fr/about/" />
<link rel="alternate" hreflang="de"      href="https://cubewise.com/de/about/" />
<link rel="alternate" hreflang="zh-Hans" href="https://cubewise.com/zh-hans/about/" />
<link rel="alternate" hreflang="x-default" href="https://cubewise.com/about/" />
```

The base URL (`https://cubewise.com`) is read from `config.js`.

## Translation Workflow

1. **Mark strings** — add `data-i18n` attributes to all translatable elements in source HTML (one-time effort, then ongoing as new content is added)
2. **Extract** — `node scripts/extract-strings.js` → produces `src/i18n/en.json`
3. **Translate** — `node scripts/translate.js --locale fr` (repeat for `de`, `zh-hans`)
4. **Human review** — translator/reviewer edits `src/i18n/{locale}.json` directly
5. **Build** — `node scripts/preview.js` or `node scripts/build.js` to see results

When new strings are added to source HTML:
- Re-run extract (preserves existing values, adds new keys)
- Re-run translate with `--locale` (skips already-translated keys, only translates new ones)
- Reviewer checks only new keys

## Out of Scope

- Blog posts and docs (Confluence-sourced) — English only
- Automatic locale detection from `Accept-Language` header
- Subdomain-based routing (`fr.cubewise.com`)
- Right-to-left language support

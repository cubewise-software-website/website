# Docs Section Design

**Date:** 2026-03-31  
**Status:** Approved

## Goal

Build a unified docs section at `/docs/` pulling content from 10 Confluence spaces across 5 products, with a landing page, client-side search, sidebar filter pills, breadcrumbs, and "on this page" navigation.

---

## Space Configuration

Add `product` and `docType` fields to each docs space entry in `config.js`:

```js
{ key: 'AIAC1', section: 'docs', template: 'docs', isDocsSpace: false, product: 'arc',          docType: 'installation' },
{ key: 'AUM',   section: 'docs', template: 'docs', isDocsSpace: false, product: 'arc',          docType: 'manual' },
{ key: 'AIAC',  section: 'docs', template: 'docs', isDocsSpace: false, product: 'arc-plus',     docType: 'installation' },
{ key: 'AUM1',  section: 'docs', template: 'docs', isDocsSpace: false, product: 'arc-plus',     docType: 'manual' },
{ key: 'PIC',   section: 'docs', template: 'docs', isDocsSpace: false, product: 'powerconnect', docType: 'installation' },
{ key: 'PUM1',  section: 'docs', template: 'docs', isDocsSpace: false, product: 'powerconnect', docType: 'manual' },
{ key: 'PIAC',  section: 'docs', template: 'docs', isDocsSpace: false, product: 'pulse',        docType: 'installation' },
{ key: 'PUM',   section: 'docs', template: 'docs', isDocsSpace: false, product: 'pulse',        docType: 'manual' },
{ key: 'SIAC',  section: 'docs', template: 'docs', isDocsSpace: false, product: 'slice',        docType: 'installation' },
{ key: 'SUM',   section: 'docs', template: 'docs', isDocsSpace: false, product: 'slice',        docType: 'manual' },
```

`isDocsSpace` is set to `false` — label injection is handled explicitly via `product` and `docType` instead of the space key.

## Labels

In `fetch.js` / `render.js`, when processing a docs space, auto-inject two labels onto every page:
- `product` value (e.g. `arc-plus`)
- `docType` value (e.g. `installation`)

These labels are passed through to the rendered page object and written as `data-product` and `data-doc-type` attributes on each sidebar link.

---

## URL Structure

All docs pages render at a flat path: `/docs/{slugify(page.title)}/`

`buildPath` in `render.js` for `section === 'docs'` returns: `docs/${slugify(page.title)}`

No product or type prefix in the URL.

---

## Build: Search Index

During build, after all docs pages are rendered, `build.js` writes `dist/assets/search-index.json`:

```json
[
  {
    "title": "Installing Arc",
    "slug": "installing-arc",
    "excerpt": "First 150 characters of plain text stripped from body HTML...",
    "product": "arc",
    "docType": "installation"
  },
  ...
]
```

The excerpt is generated by stripping HTML tags from the rendered body and taking the first 150 characters.

---

## Build: Sidebar Data

`build.js` also generates the sidebar HTML passed to `{{sidebar}}` in `docs.html`. The sidebar contains all docs pages grouped by product, then by docType, with `data-product` and `data-doc-type` attributes on each group and link:

```html
<div class="sidebar-product" data-product="arc">
  <div class="sidebar-group" data-doc-type="installation">
    <h4>Arc — Installation &amp; Config</h4>
    <ul>
      <li><a href="/docs/installing-arc/" data-product="arc" data-doc-type="installation">Installing Arc</a></li>
      ...
    </ul>
  </div>
  <div class="sidebar-group" data-doc-type="manual">
    <h4>Arc — User Manual</h4>
    <ul>...</ul>
  </div>
</div>
```

## Docs Landing Page (`/docs/index.html`)

Static hand-coded page at `src/pages/docs/index.html`.

**Structure:**
1. **Search bar** — prominent, centered input with placeholder "Search docs…". On focus, loads `search-index.json` once. As user types (debounced 150ms), filters index by title + excerpt match and renders top 5 results as a dropdown below the input. Each result shows title, product badge, and docType. Clicking a result navigates to `/docs/{slug}/`.

2. **Product card grid** — 5 cards (Arc, Arc+, PowerConnect, Pulse, Slice), 3-column layout on desktop, 2-column on tablet, 1-column on mobile. Each card contains:
   - Product logo (`/assets/logos/Products/{Product}/`)
   - Product name
   - Two links: "Installation & Configuration" → `/docs/?product={product}&type=installation` and "User Manual" → `/docs/?product={product}&type=manual`
   - These links navigate to the full docs listing page filtered by that product/type (future: could deep-link into sidebar filter state)

`{{breadcrumb}}` is rendered at build time: `Docs › {Product display name} › {docType display name} › {title}`

## JavaScript

**`src/assets/js/docs-filter.js`** — sidebar filter pills:
- On pill click, sets active state, reads `data-filter-product` and `data-filter-type` values
- Iterates `.sidebar-product[data-product]` and `.sidebar-group[data-doc-type]` elements, toggling `display: none` based on active filter combination
- "All" pill resets all groups to visible

**`src/assets/js/docs-search.js`** — landing page search:
- Fetches `/assets/search-index.json` once on input focus
- On input (debounced 150ms), filters index: title match weighted higher than excerpt match
- Renders top 5 results into a `.docs-search-results` dropdown
- Keyboard navigation (arrow keys, enter, escape) on the dropdown
- Closes dropdown on outside click

**`src/assets/js/docs-toc.js`** — "on this page" anchors:
- On DOMContentLoaded, finds all `h2` and `h3` inside `.docs-content`
- Builds a `<ul>` of anchor links and injects into `#docs-toc`
- If fewer than 2 headings found, hides `#docs-toc`

## CSS

New rules added to `main.css`:

- `.docs-filter-pills` — flex column, gap between rows
- `.filter-row` — flex wrap, gap between pills
- `.filter-pill` — small pill button, outlined style, active state uses `var(--orange-gold)` border + text
- `.docs-breadcrumb` — small muted text, `›` separator
- `.docs-toc` — sticky positioned within content column, small font, muted links
- `.docs-search-results` — absolute dropdown, white card, border, shadow, max 5 items
- `.docs-product-grid` — 3-column CSS grid on landing page
- `.docs-product-card` — white card, border, hover shadow, product logo + name + two links

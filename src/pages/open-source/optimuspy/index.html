Dimension order matters more than you think

In IBM Planning Analytics / TM1, the order of dimensions in a cube directly affects RAM consumption and query performance. The wrong order can result in significantly higher memory usage and slower calculations — but finding the right order manually across many cubes is impractical.

  • Dimension order directly impacts RAM usage and query speed
  • The optimal order differs for every cube based on its data distribution
  • Manual analysis across a large TM1 server is time-consuming and error-prone
  • OptimusPy automates the entire analysis and recommendation process

Measure, compare, and recommend

OptimusPy tests permutations of dimension order for each cube, measuring actual RAM consumption and query performance for each combination. It then identifies and reports the optimal ordering — backed by real measurements, not guesswork.

  • Tests dimension order permutations against your live TM1 data
  • Measures RAM usage and query execution time for each combination
  • Identifies the optimal order per cube automatically
  • Generates a clear report of recommended changes

Built on TM1py

OptimusPy is powered by TM1py — the open-source Python framework for IBM Planning Analytics. It connects directly to your TM1 server, performs its analysis, and outputs recommendations without requiring any changes to your environment.

  • Uses TM1py for all TM1 server interactions
  • Connects securely to your TM1 instance — no additional infrastructure needed
  • Works with IBM Planning Analytics on-premises and cloud deployments
  • Simple configuration — point it at your server and run
